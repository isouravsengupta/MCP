# SPI Sales Data Skill

**Purpose:** Give Markus and the GSI leadership team natural-language access to
Snowflake Sales Programme (SPM) data directly from Slack — without opening a CRMA dashboard.
Results match the Global SPM Performance and Data Accuracy Hub dashboard calculations
because all SQL is derived from the same CRMA business logic files.

---

## Data available

All metrics come from the `{DB}.AIO` schema. The key Snowflake tables are:

| Table | What it contains |
|---|---|
| `GSP_SPM_ACV_PERM` | Opportunity-level ACV influenced by Sales Programmes |
| `GSP_SPM_PIPEGEN_PERM` | Opportunity-level pipeline generated by Sales Programmes |
| `GSP_SPM_NNAOV_PERM` | AOV Protection / NNAOV metrics per programme |
| `GSP_SPM_PROGRAM` | Programme-level summary: ACV, pipegen, close rate, C&C |
| `GSP_SPM_TARGET_ACCTS_PERM` | Target accounts per programme |
| `GSP_SPM_PERFORMANCE_QA` | QA / Data Accuracy Hub metrics per programme |

---

## When to use this skill

Use when a user asks about:
- How much ACV have Sales Programmes influenced this quarter?
- What is the pipeline generated by programmes in EMEA?
- Which programmes have the best close rates?
- How many target accounts does programme X have?
- What is our AOV Protection rate?
- How many programmes are active in APAC right now?
- Show me the SPM performance summary for Q2FY27

---

## Deterministic execution sequence — always follow in order

### Step 1 — Identify the metric and filters

From the question, identify:
- **Metric ID** (see list below)
- **Filters**: region, programme name, programme type, programme status, fiscal quarter

### Step 2 — Call `list_crma_metrics`

Always call this first to confirm the exact `metric_id` string and available filter keys.

### Step 3 — Call `run_metric_query`

```
run_metric_query(
  metric_id = "<id from step 2>",
  filters   = { "region": "EMEA", "fiscal_flip_qtr": "Q2FY27" }
)
```

Available filter keys for SPM metrics:
- `region` — e.g. `EMEA`, `AMER`, `APAC`, `JAPAN`, `LATAM`
- `segment` — e.g. `ENTR`, `CMRCL`, `ESMB` (maps to `BB_DRVD_SEG_MACRO_NM_DISPLAY`)
- `program_name` — exact programme name
- `program_id` — Salesforce Campaign/Programme ID (18-char)
- `program_type` — programme type classification
- `program_status` — e.g. `Active`, `Completed`
- `program_owner` — programme owner name
- `fiscal_flip_qtr` — fiscal quarter when opportunity was created/flipped, e.g. `Q2FY27`
- `fiscal_close_qtr` — fiscal quarter when opportunity closed, e.g. `Q2FY27`
- `fiscal_flip_year` — e.g. `FY27`
- `fiscal_close_year` — e.g. `FY27`

### Step 4 — Format and respond

- Lead with the headline number (conclusion-first).
- Include the fiscal period and filter scope.
- If multiple values, use a compact table.
- Never invent numbers. If no rows returned, say so and suggest broadening the filter.

## Available metric IDs

| metric_id | What it answers |
|---|---|
| `spm_total_acv` | Total ACV influenced by programmes ($) |
| `spm_acv_opp_count` | Number of ACV opportunities influenced |
| `spm_total_pipegen` | Total pipeline generated by programmes ($) |
| `spm_pipegen_opp_count` | Number of pipeline opportunities generated |
| `spm_nnaov_protection` | Total AOV protected by programmes ($) |
| `spm_aov_protection_rate` | Average AOV protection rate (%) |
| `spm_programme_metrics_summary` | Programme-level card: ACV, pipegen, close rate, C&C |
| `spm_programme_count` | Number of distinct active programmes |
| `spm_close_rate` | Average close rate across programmes (%) |
| `spm_target_account_count` | Number of target accounts in a programme |
| `spm_performance_qa_summary` | QA/accuracy hub: product tags, target accounts, influenced opps |

---

## Example interactions

**"How much ACV have EMEA programmes influenced in Q2FY27?"**

```
run_metric_query(
  metric_id = "spm_total_acv",
  filters   = { "region": "EMEA", "fiscal_close_qtr": "Q2FY27" }
)
```
Reply: "EMEA Sales Programmes influenced **$142M ACV** in Q2FY27."

---

**"What is the pipeline generated by Active programmes in AMER this year?"**

```
run_metric_query(
  metric_id = "spm_total_pipegen",
  filters   = { "region": "AMER", "program_status": "Active", "fiscal_flip_year": "FY27" }
)
```

---

**"How many programmes are running in APAC?"**

```
run_metric_query(
  metric_id = "spm_programme_count",
  filters   = { "region": "APAC", "program_status": "Active" }
)
```

## Constraints

- SELECT queries only — no INSERT / UPDATE / DELETE / DDL ever.
- Always include at least one filter (region or fiscal period) on large tables to avoid full scans.
- Max 200 rows per response — the metrics are pre-aggregated so this is rarely a constraint.
- Never expose Snowflake credentials, database names, or internal error stack traces.
- If the user asks for data not in the metric catalog, use `list_tables` then `describe_table` to explore — confirm with the user before running an ad-hoc query.
